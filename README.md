## Getting Started

To run this Vite project locally:

```bash
git clone https://github.com/daud-hayat-8183/dawood-hayat-portfolio.git
cd dawood-hayat-portfolio
npm install
npm run dev
```

*Note: Environment files containing secrets should not be committed to this repository.*

## Security and Environment Variables

This repository does not contain production API keys, email-provider credentials, database passwords, private environment files, or server-only secrets.

Copy `.env.example` to `.env.local` for local configuration. Never commit `.env.local`, API keys, tokens, passwords, private keys, or service-role database credentials.

- Client-side public variables must be limited.
- Sensitive keys must remain server-side.
- Contact form secrets should be configured in Vercel or another deployment platform.
- Do not expose Resend, Supabase service-role, Turnstile secret, Gmail credentials, or database URLs in frontend code.

## Deployment

- GitHub stores source code.
- Vercel is recommended for production deployment.
- Vercel should be connected to this repository.
- Environment variables must be configured in Vercel project settings.
- Do not use GitHub Pages for secure server-side contact API functionality.
